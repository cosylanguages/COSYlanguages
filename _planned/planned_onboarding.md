# Planned onboarding flows

## Current situation
There is no visible flow for:
- A new teacher to sign up and get a teacher code
- A new student to get their access code from their teacher
- A new free user to understand what they can access

## Planned: Teacher onboarding
| Step | Status | Notes |
|------|--------|-------|
| Contact/enquiry form on homepage | Not built | Even a Google Form embed would work |
| James creates teacher code manually | Manual | Acceptable for small scale |
| Welcome email with code and instructions | Not built | Could be a mailto template |
| Teacher portal intro / help page | Not built | First-time teacher sees a guide |

## Planned: Student onboarding
| Step | Status | Notes |
|------|--------|-------|
| Student receives code from teacher | Manual | Acceptable |
| Student enters code on portal | Partially built | Stage 9 addresses this |
| Student sees placement quiz suggestion | Not built | Not sure of your level? Take the quiz |
| Student reaches dashboard | Not built | Stage 9 addresses this |

## Planned: Free user onboarding
| Step | Status | Notes |
|------|--------|-------|
| Homepage clearly explains free access | Unclear | Needs a dedicated section |
| Free user can browse grammar and vocabulary | Partial | |
| Free user can play some games | Partial | |
| Free user sees a gentle prompt to get a code | Not built | Not pushy, just informative |

## Planned: Placement quiz
| Detail | Notes |
|--------|-------|
| 10-15 questions per language | Mix of grammar, vocabulary, reading comprehension |
| Fully in the target language | No translation — emoji and visual aids for navigation only |
| Result: suggested level A1-C2 | With brief explanation in the target language |
| Stored in localStorage | Student sees suggested level on their dashboard |
